# RP Lounge Manager — 設計規格文件

**日期：** 2026-04-07
**專案目錄：** `tools/`

---

## 1. 專案概述

為 FFXIV RP 招待所設計的內部營運管理工具，提供三大核心功能：點台倒計時、香檳消費排行記錄、員工薪資自動計算。

主要使用情境：管理者在桌機上邊玩 FFXIV 邊以第二螢幕或切換分頁操作；老闆或其他成員亦可從手機查看。

---

## 2. 技術架構

| 項目 | 選擇 |
|------|------|
| 前端框架 | React + Vite |
| 樣式 | Tailwind CSS |
| 資料庫 | Firebase Firestore |
| 驗證登入 | Firebase Auth（Email 登入） |
| 部署 | Cloudflare Pages |

```
使用者（瀏覽器）
     ↓
React 前端（PWA，Cloudflare Pages）
     ↓
Firebase Firestore（即時同步雲端資料庫）
```

---

## 3. 使用者權限

單一權限設計：登入後即可使用所有功能。日後有需要再加入角色區分。

## 4. 整體介面結構

側邊導覽列（Dashboard）佈局：

- **桌面：** 左側固定 nav，右側主內容區

## 5. 功能規格

### 5.1 點台計時

**出勤選擇區（上方）**

- 橫向排列所有員工卡片（大頭貼 + 名字）
- 點選切換今日出勤狀態；出勤中亮起，未出勤灰暗

**員工 Grid（主區域）**

僅顯示今日出勤員工。每張卡片包含：

| 狀態 | 顯示內容 |
|------|---------|
| 未計時 | 大頭貼、名字、三個啟動按鈕 `[10] [20] [30]`（分鐘） |
| 計時中 | 大頭貼、名字、進度條、MM:SS 倒數 |
| 時間到 | 卡片重置回「未計時」狀態 |

**視覺警示：**

| 剩餘時間 | 顏色 |
|---------|------|
| > 1 分鐘 | 綠色 |
| ≤ 1 分鐘 | 黃色 |
| 時間到 | 紅色（短暫閃爍後重置） |

**節數記錄：**

計時器自然倒數到 0 後，靜默自動記錄 +1 節至當日該員工的計時記錄，無需手動確認。卡片直接重置。

---

### 5.2 香檳記錄

**新增表單（上方）**

欄位：客人名稱、員工（下拉選單）、單支數量（+/- 按鈕）、香檳塔數量（+/- 按鈕）

**累加邏輯：**

- 相同客人 + 相同員工 → 自動累加
- 相同客人 + 不同員工 → 各自獨立一筆

**排行榜（主區域）**

依客人總消費金額降序排列，即時更新：

```
排名  客人      總金額        明細
#1   艾倫      290,000 Gil   小花×3支 / 小玉×1塔
#2   薩菲羅斯  150,000 Gil   小玉×5支
```

**歷史查詢：** 右上角日期選擇器，預設今日，可切換查看過去記錄。

**金額計算：**

- 單支香檳：30,000 Gil
- 香檳塔：200,000 Gil

---

### 5.3 薪資計算

**選擇區：** 員工下拉 + 日期選擇器（預設今日）

**自動帶入：**

- 當日節數（從計時器記錄讀取）
- 當日香檳明細（從香檳記錄讀取，僅計算該員工相關筆數）

**手動輸入：**

- 上班時數（小時，手動填寫）
- 節數可手動覆蓋修改

**計算公式：**

```
保底     = 上班時數 × 50,000 Gil
香檳抽成 = (單支 × 30,000 + 塔 × 200,000) × 50%
點台抽成 = 節數 × 40,000 × 40%
當日薪資 = 保底 + 香檳抽成 + 點台抽成
```

**明細輸出範例：**

```
保底      4 小時 × 50,000         = 200,000 Gil
香檳抽成  (3支×30,000+1塔×200,000) × 50% = 145,000 Gil
點台抽成  8節 × 40,000 × 40%     = 128,000 Gil
─────────────────────────────────────────────────
當日薪資                          = 473,000 Gil
```

**歷史記錄：** 頁面下方列出過去存檔記錄，可依員工或日期篩選。

---

### 5.4 員工管理

| 欄位 | 說明 |
|------|------|
| 名字 | 必填 |
| 大頭貼 | 上傳圖片或填入圖片網址 |
| 啟用狀態 | 停用的員工不出現在計時器 Grid 與下拉選單 |

操作：新增、編輯、刪除員工。

## 6. Firestore 資料結構

```
employees/{id}
  name: string
  avatarUrl: string
  active: boolean

sessions/{YYYY-MM-DD}/timers/{id}
  employeeId: string
  durationMinutes: number
  completedAt: timestamp

sessions/{YYYY-MM-DD}/champagne/{id}
  guestName: string
  employeeId: string
  bottles: number
  towers: number
  createdAt: timestamp

sessions/{YYYY-MM-DD}/salary/{employeeId}
  hoursWorked: number
  sessionCount: number
  bottles: number
  towers: number
  basePay: number
  champagnePay: number
  timerPay: number
  total: number
  savedAt: timestamp
```

所有營運資料掛在日期節點下，歷史查詢直接讀取對應日期的整個 `sessions/{YYYY-MM-DD}`。
